=== LAYER 2: Sequential Processing ===

Sau Canary, tat ca cac record (bao gom ca canary violation) deu di qua
MemStreamScoringOperator. Record nao co canary_violation=True se co
anomaly_score cao hon do da bi danh gia la vi pham.

CHIA NHANH CANARY:

   canary_stream (has_violation=True)
       -> MetaAggregator (voi anomaly_score mac dinh cao)
       -> IEC

   clean_stream (has_violation=False)
       -> ExtractNeighborhoodFunction
       -> MemStreamScoringOperator (AE + Memory scoring)
       -> MetaAggregator

MemStreamScoringOperator (MapFunction, moi record goi 1 lan):

   STEP 1: 34D Feature Extraction (MemStreamVectorizer.transform)
       Record dict -> numpy array 34 chieu:
       - 6 raw: trip_distance, dur_min, fare_amount, passenger_count, total_amount, speed_mph
       - 3 derived: fare_per_mile, fare_per_min, fare_per_pax
       - 3 temporal: pickup_hour, pickup_dow, is_weekend
       - 4 spatial: PU_Grid_X/Y, DO_Grid_X/Y (16x17 grid)
       - 4 normalized: fare_per_mile_norm(/2.5), fare_per_min_norm(/0.67), speed_norm(/12), pax_per_mile
       - 5 cyclic: hour_sin, hour_cos, dow_sin, dow_cos, dist_squared
       - 5 one-hot: ratecode_1...ratecode_5
       - 4 misc: is_night, log_fare, log_distance, inter_borough

   STEP 2: Normalize (StandardScaler fitted during warmup)
       x_norm = (x - mean) / (std + 1e-8)
       mean/std duoc tinh tu ultra-clean training data, frozen sau warmup

   STEP 3: AE Encoding
       z = ae.encode(x_norm)  # [34] -> [68] -> [34] (AE output, NOT hidden layer)
       PyTorch: z = ReLU(Linear(x_norm, 34, 68))

   STEP 4: kNN Scoring on Memory Buffer
       Memory buffer: FIFO circular, 50,000 vectors x 34 chieu = ~6.8MB per neighborhood
       Tim K=10 neighbors gan nhat trong memory (L1 distance):
         dist_i = sum(|z - memory[i]|) for i in 0..N-1
         top_k = 10 khoang cach nho nhat
       Score = sum(top_k_distances)  # voi gamma=0.0, chi lay neighbor gan nhat

   STEP 5: ContextBeta Ratio Scoring
       Tra cuu threshold theo 4 chieu:
         - neighborhood (10 gia tri)
         - time_window (4: night/morning/afternoon/evening)
         - day_type (3: weekday/weekend/holiday)
         - trip_type (7: ratecode categories)
       Cell ID = (is_special << 2) | (is_night << 1) | is_weekend
       Tra cuu threshold[neighborhood][cell_id] -> beta
       Normalized_score = raw_score / beta

   STEP 6: Anomaly Decision
       Neu normalized_score >= 1.0 -> is_anomaly = True
       Neu < 1.0 -> is_anomaly = False

   STEP 7: Conditional Memory Update
       Neu is_anomaly = False (score < beta):
         z_detach = z.detach()  # Cat gradient
         memory.update(z_detach)  # Insert vao FIFO buffer
       Neu is_anomaly = True:
         KHONG update memory (tranh contamination)

   STEP 8: Retrain Signal Polling (Polling Interval = 60 records)
       - Poll MinIO cadqstream-drift/iec/retrain/{nb}/
       - Neu co signal moi (timestamp > last_checked):
         + Reset memory buffer for affected neighborhood
         + Clear warmup buffer
         + Re-trigger warmup sequence
       - HMAC verification (SHA256) tren moi signal

   STEP 9: Beta Polling (Cache TTL = 1.1s)
       - Poll MinIO cadqstream-drift/iec/betas/{nb}/
       - Neu co beta moi -> cap nhat local ContextBeta
       - HMAC verification

   STEP 10: Append to record
       record['anomaly_score'] = normalized_score
       record['is_anomaly'] = is_anomaly
       record['neighborhood'] = neighborhood_name
       record['score_components'] = {...}

SequentialFinalDecisionFunction (MapFunction):
   - Neu has_violation=True (canary) -> final_decision = 'CANARY_VIOLATION'
   - Neu is_anomaly=True (ML) -> final_decision = 'ANOMALY'
   - Con lai -> final_decision = 'PASS'

=== LAYER 4: Intelligent Evolution Controller ===

Nhan meta-metrics tu Layer 3, monitor drift, quyet dinh adaptation strategy.

IECOperator (MapFunction, moi meta-record goi 1 lan):

   STEP 1: Update 60 ADWIN Instances
       MultiInstanceADWIN: 10 neighborhoods x 6 metrics = 60 instances
       Moi instance nhan 1 gia tri meta-metric:
         ADWIN instance.update(value) -> drift_detected (True/False)
       ADWIN logic:
         - Duy tri sliding window W
         - Tach W thanh W0 (cu) va W1 (moi)
         - Tinh diff = |mean(W0) - mean(W1)|
         - Tinh threshold = sqrt((1/2m) * ln(4n/delta))
         - Neu diff > threshold -> DRIFT DETECTED
           + Cat nua cu cua W (W0)
           + W = W1
           + Tra ve True

   STEP 2: DriftAggregator
       - add_drift(neighborhood, metric) cho moi drift event
       - assess_drift_severity() -> NONE/LOW/MODERATE/HIGH
         NONE: 0 drifts trong rolling window 10
         LOW: 1-2 drifts
         MODERATE: 3-5 drifts
         HIGH: 6+ drifts
       - predict_strategy(severity):
         NONE/LOW/MODERATE -> do_nothing
         HIGH -> quick_retrain

   STEP 3: Check 3 Hard Triggers (bat buoc quick_retrain bat ke severity)
       Trigger A (ADWIN Drift Days):
         - Neu ADWIN drift 7+ consecutive days -> quick_retrain
       Trigger B (Anomaly Rate):
         - Neu anomaly_rate > 15% trong 3+ consecutive days -> quick_retrain
       Trigger C (kNN Distance):
         - Neu avg_knn_distance > 2x baseline -> quick_retrain
         - Baseline = mean of first 50 post-retrain kNN distances

   STEP 4: CircuitBreaker Gate
       - should_allow_action() -> True/False
         CLOSED: cho phep action
         OPEN: neu > 10 consecutive actions -> block 5 phut
         HALF_OPEN: sau cooldown, cho phep 3 actions de test
       - Neu circuit OPEN -> tra ve do_nothing (coi nhu binh thuong)

   STEP 5: Execute Strategy
       do_nothing:
         - Khong lam gi ca
         - Khong increment circuit breaker counter

       quick_retrain:
         - Ghi retrain signal to MinIO:
           Path: s3://cadqstream-drift/iec/retrain/{neighborhood}/{timestamp}.json
           Signal: {neighborhood, severity, triggers, timestamp, hmac}
         - HMAC-SHA256 signed (MEMSTREAM_MODEL_SIGNING_KEY)
         - record_action() -> tang counter
         - on_action_success() -> reset counter, reset kNN baseline
         - on_action_failure() -> tang counter, co the trip()

   STEP 6: Emit Metrics to Prometheus (cadqstream-metrics:9250)
       - iec_decisions_total{strategy=...}
       - iec_drift_detected_total
       - iec_confidence_score
       - iec_circuit_breaker_state

   STEP 7: Append to record
       record['drifts_detected'] = [...drift events...]
       record['iec_strategy'] = 'do_nothing' hoac 'quick_retrain'
       record['iec_confidence'] = 0.0 - 1.0
       record['severity'] = severity_level

1. TAI SAO CAN FEATURE ENGINEERING?

Raw data NYC Taxi co nhieu van de:
- 5 columns cung NULL 4,091,232 records (9.94%) -> systematic failure
- Negative fares (1.78%), zero distance (1.89%), impossible speeds (max 5M mph)
- Ratecode variance: Standard $15.42, JFK $52.18, Newark $68.44
- Bimodal fare distribution (mode $8-12 in-borough, $50-60 airport)

Feature engineering giai quyet 3 van de:
a) SCALE: $50 fare la binh thuong cho JFK nhung bat thuong cho 0.5-mile trip
   -> Ratio features (fare_per_mile, fare_per_min) normalize for scale
b) CONTEXT: Fare distribution khac nhau theo thoi gian, dia diem, ratecode
   -> Temporal, spatial, cyclic features capture context
c) ANOMALY SIGNALS: Mot so fraud chi phat hien duoc khi xet nhieu features cung luc
   -> multivariate analysis qua AE

3. TAI SAO TUNG FEATURE CU THE?

**Ratio features (6,7,8):** Giam coefficient of variation 10-100 lan.
   $50 fare cho 0.5-mile trip co fare/mile = $100/mile -> bat thuong
   $50 fare cho 10-mile trip co fare/mile = $5/mile -> binh thuong
   Ratio features doc lap voi gia tri tuyet doi cua fare.

**Normalized features (16,17,18):** Chia cho empirical median de reduce variance.
   Denominator 2.5 = median NYC fare/mile
   Denominator 0.67 = median NYC fare/minute
   Denominator 12.0 = median NYC speed (mph)

**Cyclic features (20-24):** Bao toan proximity:
   Hour 23 va hour 0 cach nhau 1 hour nhung Euclidean distance = 1.0
   Cyclic encoding: sin/cos(2pi*hour/24) cho khoang cach = 0.26 (gan nhau hon)
   Quay 23h -> 0h la transition binh thuong, khong phai anomaly.

**One-hot ratecode (25-29):** JFK co dinh $52, Newark co dinh $68.
   Mot $55 fare la binh thuong cho JFK nhung bat thuong cho Standard.
   One-hot cho phep model phan biet context.

**Log transforms (31,32):** Compress high-fare range, giam outlier influence.
   Fare $335,544 co log(1+335544) = 12.72, gần voi $100 (log=4.62)
   Khong log: $335,544 >> $100, se dominate gradient.

1. ADWIN (ADAPTIVE WINDOWING)

**Muc dich:** Phat hien concept drift trong streaming data.

Motivation: Neu phan phoi du lieu thay doi (drift), model hoc tu qua khu se
khong con chinh xac nua. Can phat hien drift de trigger retraining.

**Thuật toán ADWIN (chi tiet mathematical):**

ADWIN duy tri mot sliding window W chua nhung gia tri gan nhat.

Khi mot gia tri moi x den:
1. Append x vao cuoi W
2. Neu len(W) > max_window (500): trim oldest entries
3. Tach W thanh 2 sub-windows: W0 (older) va W1 (newer)
   - recent_size = min(n//4, 100)
   - W1 = W[-recent_size:] (100 phan tu cuoi)
   - W0 = W[:-recent_size] (tat ca truoc do)
4. Tinh means:
   - mean_W0 = sum(W0) / len(W0)
   - mean_W1 = sum(W1) / len(W1)
5. Tinh diff = |mean_W1 - mean_W0|
6. Tinh threshold:
   - Hoeffding bound: epsilon = sqrt((1/(2*m)) * ln(2n/delta))
   - m = 2*|W0|*|W1| / (|W0|+|W1|) (harmonic mean)
   - n = len(W)
   - delta = 0.002 (confidence parameter, configurable per metric)
7. Neu diff > epsilon:
   - DRIFT DETECTED = True
   - Cat W0 ra khoi W: W = W1 (reset to newer sub-window)
   - Tra ve True
8. Neu khong: tra ve False

**Thuc te trong CA-DQStream:**

60 ADWIN instances (10 neighborhoods x 6 metrics):
- volume ADWIN (delta=0.005, 5-min window)
- null_rate ADWIN (delta=0.001, 1-min window)  <- NHAT SENSITIVE
- violation_rate ADWIN (delta=0.002, 1-min window)
- anomaly_rate ADWIN (delta=0.001, 1-min window) <- SENSITIVE
- avg_anomaly_score ADWIN (delta=0.001, 1-min window)
- delta_score ADWIN (delta=0.001, 5-min window)

Vi du: null_rate ADWIN co delta=0.001 (rat sensitive), chi can
2.9% diff la phat hien drift. Violation_rate co delta=0.002, can 4.1% diff.

Neu 1 neighborhood co drift tren 7+ consecutive days -> Trigger A fire.


2. METER (MULTI-INSTANTIATED ADWIN)

**Muc dich:** Theo doi nhieu metrics cung luc, tai nhieu locations cung luc.

METER la extended ADWIN co nhieu instances hoat dong song song.

**Cau truc trong CA-DQStream:**

MultiInstanceADWIN class quan ly 60 instances:
- 10 neighborhoods * 6 metrics = 60 SimpleADWIN objects
- Moi instance co independent window, independent state
- Drift trong Manhattan khong anh huong Brooklyn

**ADWIN-U (Unsupervised Extension):**

ADWIN goc chi hoat dong voi supervised data (co labels).
ADWIN-U mo rong cho unsupervised setting:

a) Higher-order statistics:
   - Skewness (gamma_1): phat hien asymmetric shifts
     gamma_1 = E[(X-mu)^3] / sigma^3
   - Kurtosis (gamma_2): phat hien tail-mass changes
     gamma_2 = E[(X-mu)^4] / sigma^4 - 3

b) Jensen-Shannon divergence:
   D_JS(P0 || P1) thay the mean comparison
   cho distributional-level drift detection

Trong CA-DQStream, ADWIN-U chu y:
- 6 meta-metrics la scalar summaries (da tinh tu raw data)
- ADWIN operate tren scalar metrics, khong operate tren raw features
- Delta_score = |violation_rate - anomaly_rate| / (...) la 1 meta-metric dac biet
  the hien disagreement giua Canary va Complex stage

**Tại sao dung meta-metrics thay vi raw data?**

1. Interpretability: drift tren null_rate co nghia ro rang (upstream failure)
   drift tren raw fare chi la "co thay doi" nhung khong ro gi
2. Dimensionality: Raw la 34D, meta-metrics la 6D -> 6 ADWIN instances thay vi 34
3. Statistical power: Meta-metrics la aggregate, variance thap hon raw data
4. Noise robustness: Meta-metrics loc noise, chua signal drift thuc su

1. OFFLINE PRETRAIN (truoc khi deploy)

**Muc dich:** Hoc weight cua Autoencoder va compute 4D threshold matrix
tu ultra-clean training data.

**Training data preparation (3-stage funnel):**

INPUT: 3,000,000 records (January 2024)

Stage 1: Physical Violation Filter
- Loai bo: negative fare, zero distance, impossible timestamps,
  GPS outliers (lat not in [40.4, 41.0]), ratecode-zone inconsistencies
- Con lai: 2,923,500 records (loai bo 76,500 = 2.55%)

Stage 2: IQR Outlier Removal
- Ap dung IQR 1.5x method tren fare, distance, speed, duration
- [Q1 - 1.5*IQR, Q3 + 1.5*IQR] lai acceptable
- Con lai: 2,896,900 records (loai bo 26,600 = 0.89%)

PHAN 5: MODEL AE + KNN HOAT DONG THE NAO

1. AUTOENCODER (DENOISING AUTOENCODER)

**Kien truc:**

Encoder: x (34D) -> Linear(34, 68) -> ReLU -> h (68D)
Decoder: h (68D) -> Linear(68, 34) -> ReLU -> x_hat (34D)

Tied weights: W_decoder = W_encoder^T (giup giam 50% parameters)

**Training:**

Input: x (34D) duoc corrupt voi noise:
  x_corrupt = x + epsilon, epsilon ~ N(0, 0.1^2)

Loss: L = MSE(x, x_hat) = sum((x_i - x_hat_i)^2)

Optimizer: Adam, lr=1e-3
Batch size: 256
Early stopping: 20 epochs patience

**Tai sao Denoising?**

Neu AE chi hoc identity mapping (x_hat = x), latent space vo nghia.
Noise buoc AE hoc feature extraction that su, khong chỉ copy.

Example:
- Normal trip: x gần với manifold -> sau denoising van gần x_hat
- Anomaly: x xa manifold -> sau denoising van xa x_hat (high loss)

**Inference (chi dung Encoder, KHONG dung Decoder):**

Test time: z = encoder(x_norm)  # [34] -> [68] -> [34]

Decoder chi can cho training. Inference chi can encoder output z.

**Tai sao dung AE output (34D), khong phai hidden layer (68D)?**

MemStream paper (Bhatia et al., 2022) Proposition 2:
- Neu dùng hidden layer z (68D), ton tai "null-space anomalies"
  (vectors far from origin but close to hyperplane) -> false negatives
- AE output z_hat (34D) khong co null-space, an toan hon


2. L1 K-NN SCORING

**Memory Module:**

FIFO circular buffer:
- Capacity: 50,000 vectors
- Moi vector: 34D float32 = 136 bytes
- Total: 50,000 * 136 = 6.8 MB per neighborhood

Insert: Them z_detach vào vi tri hien tai, tang pointer mod N
Read: Lay tat ca vectors tu buffer

**kNN Scoring Algorithm:**

Input: z (34D encoded vector moi)
Memory: M (N x 34D, N = 50,000)

Step 1: Tinh L1 distance toi tat ca N vectors:
  dist_i = sum(|z[j] - M[i,j]|) for j=0..33, i=0..N-1
  [NumPy broadcasting: z[None,:] - M = (N, 34), sum axis=1 = (N,)]

Step 2: Tim K=10 khoang cach nho nhat:
  top_k_indices = argpartition(dist, K)[:K]  # O(N) partition
  top_k_distances = dist[top_k_indices]
  top_k_distances.sort()

Step 3: Tinh anomaly score:
  score = sum(gamma^i * top_k_distances[i]) for i=0..K-1

  Voi gamma = 0.0:
  score = top_k_distances[0]  # Chi lay gan nhat

**Tai sao gamma = 0.0?**

MemStream paper: Gamma decay de truong hop neighbor i xa cung contribute,
nhung gap nguy co "memory poisoning" (anomaly xa nhung van contribute).

gamma = 0.0 la "fresh anomaly dominates": chi neighbor gan nhat count.
Thiet ke co y: pattern moi nhat la indicator manh nhat cua current concept.

**Tai sao dung L1 (Manhattan), khong phai L2 (Euclidean)?**

1. Outlier robustness: L2 penalizes quadratically (a^2), L1 linearly (a)
   Mot coordinate lon trong L2 se dominate, trong L1 thi khong
2. High-D: L2/E[L1] ~ sqrt(d) for random vectors, L1 preserve interpretability
3. BLAS: L1 = sum(|a-b|) la 34 additions + 34 comparisons, L2 = sum((a-b)^2)
   thi 34 multiplications + 34 additions, L1 nhanh hon


3. SCORING PIPELINE (end-to-end)

Record dict
    |
    v
34D Vectorizer.transform(record)
    |
    v
[34] numpy array
    |
    v
Normalize: (x - mean) / (std + 1e-8)

else: NORMAL

  - MODERATE: 3-5 drifts
  - HIGH: 6+ drifts

Trigger B: Anomaly Rate > 15% for 3+ days
  - Track history cua anomaly_rate per neighborhood
  - Neu 3 consecutive days > 15% -> force quick_retrain
  - Gap 1-2 days below threshold -> reset counter

Trigger C: kNN Distance > 2x Baseline

  - -> Memory buffer khong con dai dien cho current concept

1. ANOMALY TYPES

**Type 1: Schema Violations (Layer 1)**

**Type 2: Canary Violations (Layer 1-2)**

- Traffic manipulation: prolonged duration cho short distance
- Payment inflation: tip chiếm % lon cua total
- Detection: AE + Memory + kNN (Layer 2 Complex)

2. MULTIVARIATE ANOMALY DETECTION FLOW

**Step-by-step:**

Record: {trip_distance: 0.1, dur_min: 1.0, fare_amount: 85.0,
         passenger_count: 1, total_amount: 92.0, PULocationID: 132,
         DOLocationID: 1, RatecodeID: 1, ...}

Step 1: Feature Extraction (34D)
  [0.1, 1.0, 85.0, 1, 92.0, 6.0,  # raw
   850.0, 85.0, 85.0,              # derived (fare_per_mile=850!)
   14, 3, 0,                        # temporal
   4, 7, 0, 0,                      # spatial grid
   340.0, 126.9, 0.5, 10.0,        # normalized (ratios muy cao!)
   sin(14), cos(14), sin(3), cos(3), 0.01,  # cyclic

Step 2: Normalize
  x_norm = (x - train_mean) / (train_std + 1e-8)

Step 3: AE Encode
  z = encoder(x_norm)  # [34D]

Step 4: kNN in Memory
  - Normal 0.1-mile Manhattan trip co z_norm gan [0.1, 1, 10, ...]
  - Fraud trip (fare $85, distance 0.1) co z hoan toan khac
  - L1 distance = 127.4 (rat lon)
  - Score = 127.4 (chi lay gan nhat, gamma=0)

Step 5: ContextBeta Lookup
  - PULocationID=132 -> queens_lower (neighborhood index 2)
  - hour=14, dow=3, ratecode=1 -> cell_id = (0<<2)|(0<<1)|0 = 0 (std_day_weekday)
  - threshold[2][0] = 35.2 (Queens lower, weekday daytime)
  - Beta = 35.2

Step 6: Normalized Score
  normalized_score = 127.4 / 35.2 = 3.62
  -> 3.62 >= 1.0 -> ANOMALY

Step 7: Memory Decision
  score (127.4) >= beta (35.2) -> is_anomaly = True
  -> KHONG update memory (vi la anomaly)

3. VI DU CAC LOAI ANOMALY

**Example 1: Meter Tampering (Easy)**
Record: distance=0.5, duration=2, fare=$30, total=$35
  fare_per_mile = 60 (normal: $2-5)
  fare_per_min = 15 (normal: $0.5-1)
  -> Score rat cao -> ANOMALY
  Canary: tat ca rules pass (fare>0, distance>0, speed<100)
  -> Chi phat hien duoc boi ML

**Example 2: Payment Inflation (Hard)**
Record: fare=$15, tip=$12, total=$27 (tip=44% of total)
  Raw features: tat ca binh thuong
  Rate: tip/total = 0.44 (normal: 0.15-0.20)
  -> fare_per_pax, log transforms deu bi anh huong nhung khong ro rang
  -> Score > beta -> ANOMALY (nho hieu ung tren nhieu features)

**Example 3: Short-Trip Fraud (Medium)**
Record: distance=0.2, duration=1, fare=$45, zone JFK (ratecode=2)
  ratecode=2 (JFK) co mean $52.18
  nhung distance=0.2-mile voi $45 -> fare_per_mile=225 (JFK binh thuong ~$3)
  -> Score rat cao -> ANOMALY

4. FALSE POSITIVE REDUCTION (ContextBeta)

Neu KHONG co ContextBeta (global threshold $45.46):
- JFK trip ($52, 12 miles) -> score=8, threshold=$45.46 -> ANOMALY (FALSE!)
- JFK co clustering chặt: tat ca trips deu gan $52 +/- $5
- Score=8 chi gap 17% cua threshold, rat thap
- Nhung global threshold $45.46 -> 8 < 45.46 -> binh thuong (OK)

Neu co ContextBeta:
- JFK threshold = $18.3 (rat thap vi clustering)
- Same trip: score=8, threshold=$18.3 -> 8 < 18.3 -> PASS

Cong cuong: FPR giam 4.9x (38.7% -> 4.2%) voi 4D thresholds

5. DELTA SCORE (Canary vs ML disagreement)

delta_score = |violation_rate - anomaly_rate| / (violation_rate + anomaly_rate + eps)

- Neu delta_score cao: Canary va ML disagree nhieu
  -> Co the co drift hoac emerging anomaly type
- Neu delta_score thap: hai stage dong nhat
  -> System on dinh

Example:
- Canary: 5% violation (7 rules)
- ML: 8% anomaly
- delta_score = |0.05 - 0.08| / (0.05 + 0.08) = 0.23 (rat cao)
-> IEC nen monitor ky hon

6. COMPLETE PIPELINE DECISION

Record -> Layer 1 (Schema) -> Layer 2 (Canary) -> Layer 2 (AE+Memory)

Decision matrix:

| has_schema_error | has_canary_violation | is_ml_anomaly | Final Decision |
|------------------|---------------------|---------------|----------------|
| True | * | * | SCHEMA_ERROR (DLQ) |
| False | True | * | CANARY_VIOLATION (rule-based) |
| False | False | True | ANOMALY (ML-detected) |
| False | False | False | PASS (clean record) |

Luu y: Canary violation van di qua ML scoring,
nhung ML se danh gia lai voi anomaly_score cao hon.

1. TAI SAO SEQUENTIAL, KHONG DUAL BRANCH?
