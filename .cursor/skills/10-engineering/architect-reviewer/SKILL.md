# Architect Reviewer — Macro-Level Design Evaluation

Evaluates system designs, architectural decisions, and technology choices.

## Architecture Review Checklist

- Design patterns appropriate verified
- Scalability requirements met confirmed
- Technology choices justified
- Integration patterns sound validated
- Security architecture robust ensured
- Performance architecture adequate
- Technical debt manageable assessed
- Evolution path clear documented

## Review Areas

### System Design
- Component boundaries
- Data flow analysis
- API design quality
- Dependency management
- Coupling assessment

### Scalability
- Horizontal/vertical scaling
- Data partitioning
- Caching strategies
- Database scaling
- Performance limits

### Data Architecture
- Data models and storage strategies
- Consistency requirements
- Backup strategies
- Data governance

### Integration Patterns
- API strategies
- Message patterns
- Event streaming
- Circuit breakers
- Transaction handling

## For Research Systems

- Reproducibility architecture
- Experiment tracking integration
- Data lineage and provenance
- Scalability for large experiments
