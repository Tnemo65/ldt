\centerline{\bf Streaming Data Quality Monitoring}

\centerline{Le Dac Thinh}

\vspace{2cm}
{\bf Abstract}

\noindent Streaming data pipelines process millions of events per second, making real-time data quality (DQ) monitoring essential. Existing open-source DQ frameworks (Great Expectations, AWS Deequ, Soda Core) operate in batch mode, cannot adapt when data distributions shift over time, and cannot detect multivariate anomalies that require understanding inter-feature relationships.

This thesis presents \textbf{CA-DQStream}, a framework for real-time streaming data quality monitoring built on Apache Kafka and Apache Flink. The proposed method consists of two components:

\begin{enumerate}
    \item \textbf{Four-stage processing pipeline:} (1) Schema Validation with deterministic NULL checks and Flink KeyedState deduplication; (2) Business Rules with six deterministic validity checks; (3) Online Autoencoder + Memory Module for multivariate anomaly detection; and (4) ADWIN drift detection on quality meta-streams for self-monitoring.
    \item \textbf{Monitoring and storage layer:} MinIO (S3-compatible object storage) persists cleaned data, dirty data, and alerts in Parquet format; Prometheus collects metrics from all pipeline components; Grafana provides real-time dashboards for operators to visualize violation rates, anomaly scores, and drift alerts.
\end{enumerate}

Together, the pipeline processes incoming records in real time while the monitoring layer provides full observability into pipeline health, data quality metrics, and drift events.

\textbf{Case Study.} The framework is evaluated on the NYC Yellow Taxi dataset, where we identify and solve specific data quality problems: completeness violations, validity violations such as negative fares and impossible timestamps, multivariate anomalies including meter tampering and short-trip fraud, and concept drift over time. On Easy-difficulty synthetic anomalies (meter tampering, short-trip fraud, traffic manipulation, payment inflation), the Online Autoencoder + Memory Module achieves F1=0.828; across Easy/Medium/Hard difficulty levels, the weighted average F1=0.71 with FPR=2.99\%. The Online Autoencoder is warmup-trained on ultra-clean January 2024 historical baseline data (~2.86M records after 3-stage filtering) and operates online with continuous learning via FIFO memory replacement. The four-stage pipeline demonstrates substantial improvements over fixed-threshold approaches. ADWIN detects gradual drift within 24 hours on average (1440 aggregated 1-minute windows), with 93\% of drift events resolved by lightweight incremental updates without full retraining, triggering automatic threshold recalculation and memory reset when needed.
